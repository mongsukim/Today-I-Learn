# 2025-11-10 작업 일지

## 🎨 디자인 시스템 학습

### 폰트 크기 단위
- **rem vs em 차이 이해**
  - rem: 루트(html) 요소 기준, 일관된 크기
  - em: 부모 요소 기준, 상대적 크기
- **앱 개발 시 8px 단위 spacing 적용** (웹/앱 공통)
- **margin/padding도 rem 단위 사용** 권장

### Tailwind CSS
- **텍스트 크기**: text-xs(12px) ~ text-9xl(128px)
- **폰트 굵기**: font-normal(400) ~ font-bold(700)
- **디자인 시스템 구성**: 5~7개 크기, 3~4개 굵기면 충분
- **line-height(leading)**: 대부분 자동 적용, 특별한 경우만 수동 조정
- **기본값**: 아무 설정 없으면 16px(1rem), font-weight 400

### Framework7 설정
- **기본 폰트 크기 변경**: `--f7-font-size: 16px` (기본 14px → 16px)
- **Dialog border-radius 조정**: `--f7-dialog-border-radius: 8px` (기본 14px → 8px)

### 스타일 마이그레이션
- **px → rem 변환 방법 학습**
  - 계산 공식: rem = px ÷ 16
  - VSCode 정규식으로 일괄 변환 가능
- **em → Tailwind 변환**
  - 기본값과 비슷하면 text-xs, text-sm 등 사용
  - 특이한 값은 text-[1.15em] 형태로 임의값 사용
- **CSS → Tailwind 변환**
  - Tailwind v4에서 @apply 사용 시 `@reference 'tailwindcss'` 필요
  - 가능하면 Tailwind 클래스 직접 사용 권장

### 환경변수 설정
- **API 호출 URL 문제 해결**
  - `.env` 파일에 `VITE_API_BASE_URL` 설정
  - Vite는 `VITE_` 접두사 필수
  - 환경변수 변경 후 개발 서버 재시작 필요
- **Git 관리**
  - `.env` 파일 `.gitignore`에 추가
  - 이미 추적 중인 파일은 `git rm --cached` 로 제거

### 빌드 프로세스 이해
- **빌드 명령어 분석**: `tsc && vite build && node scripts/after-build.js`
  - tsc: 타입 체크만 수행
  - vite build: 실제 프로덕션 빌드
  - after-build.js: 빌드 후 추가 작업 스크립트
- **린트 검사**
  - 타입 에러만 빌드 시 체크됨
  - ESLint는 별도 실행 필요
  - 팀 프로젝트에서는 에디터 ESLint 확장 설치 권장
